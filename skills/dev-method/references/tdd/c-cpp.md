# C/C++ TDD Best Practices

## Test Framework: Unity + CMock (C) / Google Test (C++)

### Option 1: Unity + CMock (for C)

#### Setup
```bash
# Install Unity and CMock
git clone https://github.com/ThrowTheSwitch/Unity.git
git clone https://github.com/ThrowTheSwitch/CMock.git

# Directory structure
project/
├── src/
├── tests/
│   ├── test_example.c
│   └── test_example_runner.c
└── Makefile
```

#### Test Structure
```c
/* tests/test_mymodule.c */
#include "unity.h"
#include "mymodule.h"

void setUp(void) { /* Optional: runs before each test */ }
void tearDown(void) { /* Optional: runs after each test */ }

void test_mymodule_initialization(void) {
    MyModule mod;
    mymodule_init(&mod, 42);
    TEST_ASSERT_EQUAL(42, mod.value);
}

void test_mymodule_process_returns_expected(void) {
    MyModule mod;
    mymodule_init(&mod, 0);
    int result = mymodule_process(&mod, 10);
    TEST_ASSERT_EQUAL(100, result);
}

void test_mymodule_process_handles_zero_input(void) {
    MyModule mod;
    mymodule_init(&mod, 0);
    int result = mymodule_process(&mod, 0);
    TEST_ASSERT_EQUAL(0, result);
}

```

### Option 2: Google Test (for C++)

#### Setup
```bash
# CMake approach
mkdir build && cd build
cmake .. -DBUILD_TESTING=ON
make
ctest
```

#### Test Structure
```cpp
// tests/mymodule_test.cpp
#include <gtest/gtest.h>
#include "mymodule.hpp"

TEST(MyModuleTest, Initialization) {
    MyModule mod(42);
    EXPECT_EQ(mod.getValue(), 42);
}

TEST(MyModuleTest, ProcessReturnsExpected) {
    MyModule mod(0);
    int result = mod.process(10);
    EXPECT_EQ(result, 100);
}

TEST(MyModuleTest, ProcessHandlesZeroInput) {
    MyModule mod(0);
    int result = mod.process(0);
    EXPECT_EQ(result, 0);
}

TEST(MyModuleTest, ThrowsOnInvalidInput) {
    MyModule mod(0);
    EXPECT_THROW(mod.process(-1), std::invalid_argument);
}

// Mock example using Google Mock
#include <gmock/gmock.h>

class MockDependency : public Dependency {
public:
    MOCK_METHOD(int, fetch, (int id), (override));
};

TEST(MyModuleTest, UsesMockDependency) {
    MockDependency mock;
    EXPECT_CALL(mock, fetch(1))
        .WillOnce(testing::Return(42));

    MyModule mod(&mock);
    int result = mod.useDependency(1);
    EXPECT_EQ(result, 42);
}
```

## Best Practices
- **C**: Use Unity + CMock for unit tests with mocks
- **C++**: Use Google Test + Google Mock
- Test file naming: `test_<module>.c` or `<module>_test.cpp`
- Use `setUp`/`tearDown` (Unity) or `SetUp`/`TearDown` (GTest) for common setup
- Assert early, assert often
- Mock at boundaries (hardware, network, filesystem)
- Keep tests fast; use mocks for slow dependencies
- Use CMake CTest or custom Makefile for test runner
