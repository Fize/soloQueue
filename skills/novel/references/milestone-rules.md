# 里程碑机制（通用）

> 里程碑是小说的"定期存档"。每 5 章一次，把"过去 5 章的成果 + 下一阶段的钩子"一起落锤到文档。
> 里程碑不是流水账。它同时承担三种作用：**阶段总结 + 阶段落锤 + 下一段引出**。

---

## 一、里程碑的硬性规则

### 1.1 生成时机（红线）

- 每完成 **5 章** 正文后，**必须**生成一份里程碑
- 路径：`plots/milestones/vol-{N}-ch-{Y}-summary.md`（N = 卷号，Y = 本次里程碑最后一章）
- 示例：`plots/milestones/vol-1-ch-5-summary.md`、`vol-1-ch-10-summary.md`、`vol-2-ch-5-summary.md`

### 1.2 必要字段（红线 · 缺一即不合格）

一份合格里程碑必须同时包含以下**两大节**，缺任一节**不合格**：

#### A. 阶段成果（Past）

- **人物状态**：伤情、修为、关系、心境
- **资产功法清单**：底牌、道具、丹药、灵石、法宝
- **地理坐标与时间**：当前所在地、离下次大劫 / 约战还有多久
- **本五章阶段成果**：主角**解决了什么、失去了什么、拿到了什么、认清了什么**（四问必答）

#### B. 下一阶段入口（Future · 易漏项红线）

> 这一节是最容易被遗漏的。**只有 A 没有 B 的里程碑 = 不合格。**

必须明确至少一项：

- **下一目标**：主角接下来要去做什么
- **下一危机**：即将扑来的麻烦是什么
- **下一悬念**：读者要期待什么揭底
- **下一去处**：主角要往哪走
- **下一抉择**：主角即将面临什么选择

### 1.3 五章节点钩子（红线）

> 与 [chapter-structure.md](chapter-structure.md) §2.4 联动。

每逢 5 章节点，对应章节的**豹尾 + 里程碑**必须共同完成"下一阶段钩子"的挂出：

- 豹尾：通过新变量把悬念推到读者面前
- 里程碑的 B 节：明确记录这个新变量的性质、紧迫程度、回收预期

## 三、里程碑的用法（写前必读）

### 3.1 写新章节前

**前置检查强制步骤**（见 [output-spec.md](output-spec.md) §三）：

1. 读取**最新里程碑**（`plots/milestones/` 下时间戳最新的一份）
2. 摘要"人物状态 + 资产清单 + 下一阶段入口"，作为本章起点
3. 检查活跃钩子中哪些在本章需要推进 / 回收

### 3.2 跨卷过渡

- 跨卷时必须读**上一卷末尾里程碑** + 上一卷的所有未回收钩子
- 新卷第一章必须呼应上一卷末尾的"下一阶段入口"

---

## 四、常见翻车点

### 4.1 只写 A 不写 B

- 典型表现：里程碑最后一行是"就这样，这五章告一段落"
- 后果：下一位模型（或下一次会话）读了里程碑也不知道接下来写什么
- 纠正：补出 B 节的至少一项

### 4.2 流水账式里程碑

- 典型表现：把五章的情节复述一遍，没有"成果"和"代价"
- 纠正：用"四问"约束（解决了什么 / 失去了什么 / 拿到了什么 / 认清了什么）

### 4.3 与活跃钩子不同步

- 典型表现：钩子表里某条钩子状态是"推进中"，但里程碑里看不到本次推进
- 纠正：里程碑的"钩子状态快照"节必须覆盖所有活跃钩子

### 4.4 章数错位

- 典型表现：vol-1-ch-10 的里程碑里，提到的"最后一章"是第 8 章
- 纠正：严格按 5 章一档，第 5、10、15、20…… 章完成后立刻生成

## 五、自查清单（生成里程碑后必过）

- [ ] A 节齐全：人物状态、资产清单、地理时间、四问
- [ ] B 节齐全：至少一项明确的下一阶段入口
- [ ] 钩子快照与 `plots/active-hooks/` 一致
- [ ] 路径命名规范：`plots/milestones/vol-{N}-ch-{Y}-summary.md`
- [ ] 内容不是流水账，而是"成果 + 代价 + 入口"
- [ ] 本次对应的 5 章节点豹尾，是否与 B 节的"下一阶段入口"互相呼应？

任一失败 → 重写。
