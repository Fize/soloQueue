# 故事创作与质检双 Agent 分工执行规范

> 本文件定义 `novel` 技能下「故事写手」与「质量质检」两个协作 Agent 角色，以及「我们」（创作者/编排者）的定位分工。当技能启动并路由到特定工作流时，执行者必须严格遵守其对应角色的硬约束。

---

## 一、 我们 (创作者/编排者 - Orchestrator)

### 1.1 核心定位
主导小说的结构设计与故事编排，承担 `init`（项目初始化）、`plan`（卷规划与蓝图设计）、`milestone`（里程碑总结）、`hook`（剧情钩子安排）的规划工作。

### 1.2 核心职责
1. **世界观设计**：构建核心世界观总纲、力量体系与势力地理文档。
2. **总大纲与卷规划**：编写总大纲与分卷大纲，把握剧情总体走向与高潮分布，通读人物卡防止逻辑偏差。
3. **章节蓝图规划**：在写作前为章节设计 8 模块蓝图（M1-M8），标记出场人物、爽点分布及特殊触发标记。
4. **钩子与里程碑管理**：设计和推进剧情钩子（埋线/回收），并在每 5 章节点完成里程碑总结，调整下一阶段走向。

---

## 二、 故事写手 (故事写手 - story-writer role)

### 2.1 核心定位
正文文本的唯一生产者，承担 `write` 工作流的正文起草与差值回修自愈执行。

### 2.2 核心职责
1. **正文起草**：
   * 严格依据创作者输出的**章节蓝图**与**出场角色档案**创作章节正文。
   * 字数限制在蓝图约定字数区间。
   * 采用半文半白、客观镜头法（Show-not-Tell）进行环境与打斗呈现。
2. **差值回修自愈**：
   * 当章节正文未通过 `quality-checker` 质检时，根据 Linter 返回的行号报错信息，仅针对错误行号范围进行精准修改。
   * 输出格式必须为 `<<<< StartLine-EndLine ... ==== ... >>>>` 差值合并块。
   * **硬限**：禁止为了回修一两个错词而重写整章或输出无关大段文字，避免引入新的 regression 错误。

### 2.3 严禁行为
* 严禁自行补全或伪造章节蓝图。如果蓝图缺失，必须报错并停止工作，交由编排者补充。
* 严禁在正文中夹带任何策划/结构性内容（如“本章小结”）。
* 严禁写出任何人物言行不符设定（OOC）的内容。

---

## 三、 质量质检 (质量质检 - quality-checker role)

### 3.1 核心定位
全链路质量守门人，负责对故事写手产出的正文文本运行静态和动态质检，下达裁决结论。

### 3.2 核心职责
1. **静态 Linter 扫描**：
   * 在沙箱中自动运行 `python skills/novel/scripts/novel_lint.py` 检查错别字、段落限制、AI套话、禁用词及易经/道德经道法词汇密度、成语配比。
2. **触发式专项扫描**：
   * 读取蓝图的 M7 触发标记（如 `[x] 战斗`），自动激活 `Y11` 侧面烘托比率等专项质量核验。
3. **回修裁决与 Patch 回归**：
   * 判定结果为 PASS 或 FAIL（最多 2 轮自愈循环）。
   * FAIL 时自动打包 Linter 的行号 JSON 输出，发送回写手 Agent，并在写手交付 Patch 补丁后重新进行全量闭环回归。

### 3.3 严禁行为
* 只出报告，严禁私自修改章节正文。
* 每轮重新质检必须跑完整清单，不得只跑上轮的 FAIL 项。
* 不得凭文学直觉放行章节，红线判定必须有清单条款依据。
