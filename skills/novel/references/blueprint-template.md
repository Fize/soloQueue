# 强制章节蓝图模板（8 模块）

> 本文件定义一份**合格章节蓝图**必须包含的 8 个模块。少任何一个模块、模块内关键字段为空、或核心字段数量不达标，**蓝图不合格**。
> 蓝图完备性校验是**双阶段闸门**：
>
> 1. 卷规划生成新蓝图后立即校验（PASS 才能交付该批次蓝图）
> 2. 章节创作工作流的前置检查阶段再次校验（PASS 才能进入正文创作）

---

## 零、蓝图粒度原则（强制）

> **蓝图是导演的场记，不是演员的台词本。**

- 蓝图只描述**目的、节奏、场景结构与情绪走向**，不写具体对话、独白或正文句子
- **严禁**在蓝图的任何模块中出现：引号包裹的对白、人物说出的具体话语、正文级别的描写句
- 爽点、豹尾、凤头等模块只写"发生了什么、用什么手法、达到什么效果"，不预写正文
- 违规示例：`主角眼眶一热，低声道："娘……"`（正文句，不合格）
- 合规示例：`主角认出母亲笔迹，情感爆发，以身体反应代替言语表达`（方向性描述，合格）

**理由**：好的对话与句子是写作状态最佳时自然生发的。蓝图阶段预写正文，既消耗创意，又在真正创作时形成束缚，导致章节僵化。蓝图的价值在于**把方向订死，把表达留白**。

---

## 一、模块顺序与命名（固定）

一份合格蓝图**必须**按以下顺序包含全部 8 个模块：

1. 基础信息与目标
2. 场景设计
3. 情绪节奏
4. 记忆点
5. 钩子进展
6. 章节结构
7. 触发标记与写作约束
8. 上下文准备

模块顺序固定、命名固定，便于质量质检分支 (quality-editor) 自动化校验。

---

## 二、8 个模块的详细要求

### 模块 1 · 基础信息与目标

> **注**：所属卷、章、目标字数、章节标题等基础信息必须填写在文件头部的 YAML Frontmatter 中。

**必填字段**：

- **所属 Arc**：第X幕 · 幕名（第X-Y章）
- **情绪基调**：紧张 / 压抑 / 激昂 / 悬疑 / 轻松 / 虐心，可多选
- **叙事目标**：用**一句话**描述本章必须达成的核心目标。**只写一件事**，禁止塞多（如要塞多，拆章）。格式建议：`主角在 XX 场景下，通过 XX 行动，达成 XX 目标`

**违规示例（叙事目标）**：

> 主角要展现修为、要救小师妹、要查真相、要得到法宝、要升境界

**合规示例（叙事目标）**：

> 主角在洞府遭遇截杀，借助残阵反击，救出被困的小师妹并抢回母亲遗物。

### 模块 2 · 场景设计

**必填**：列出本章所有场景，每个场景含以下字段：

- **场景名**（简短命名）
- **地点**
- **出场角色**
- **核心事件**（具体发生了什么）
- **冲突类型**（选填，若有则写：人物 vs 命运 / 社会 / 他人 / 自我 / 环境，见 [conflict-design.md](conflict-design.md)）
- **推进钩子**（选填，若本场景触动活跃钩子则绑定文件名）

**校验要点**：

- 至少 **2 个场景**（极端情况可 1 个，需在 M7 注意事项中说明理由）
- 场景名、地点、出场角色、核心事件为必填；冲突类型和推进钩子为选填（日常/过渡场景可不填）
- 场景之间应有空间 / 时间 / 人物关系的切换

### 模块 3 · 情绪节奏

**必填**：至少 **3 个位置点**，每点含情绪标签、强度、内容概述。高潮章或长章可增至 5-7 点。

**基准 3 点**（极简章适用）：

| 位置    | 情绪 | 强度 1-10 | 内容概述           |
| ------- | ---- | --------- | ------------------ |
| 0-20%   | 凤头 | ?         | 开场情绪与动作     |
| 20-80%  | 猪肚 | ?         | 主体情绪走向与爆发 |
| 80-100% | 豹尾 | ?         | 收束与悬念         |

**扩展 5 点**（标准章适用）：

| 位置    | 情绪 | 强度 1-10 | 内容概述 |
| ------- | ---- | --------- | -------- |
| 0-10%   | 凤头 | ?         | 开场     |
| 10-40%  | 攀升 | ?         | 铺展     |
| 40-70%  | 爆发 | ?         | 高潮     |
| 70-90%  | 缓冲 | ?         | 余波     |
| 90-100% | 悬念 | ?         | 新变量   |

**校验要点**：

- 位置点 **< 3** → 不合格
- 情绪强度不允许全部 ≥ 8（没有张弛）
- 必须有**至少一次情绪变化**（不能从头到尾同一强度）

### 模块 4 · 记忆点

### 模块 4 · 叙事重点

**必填**：至少 **1 条叙事重点描写**，指明本章要重点展现的情节或细节要素。

- **展现内容**（如人际关系推进 / 核心情感变化 / 动作细节展现 / 关键线索掉落 / 人物高光）
- **触发事件**（什么事件或互动来展现该重点）
- **实现手法**（如旁白侧写、动作特写、对白交锋等）

**说明**：

- 本模块旨在避免章节变成空洞的剧情流水账，要求每章必须有一个核心的内容锚点（例如“展现主角在日常插科打诨中流露的性格习惯”或“通过眼神和动作描写呈现主角与师父之间的默契”）。

**校验要点**：

- 叙事重点数 **< 1** → 不合格

### 模块 5 · 钩子进展

**必填**：列出本章"引入 / 推进 / 回收"的每个钩子，每条绑定 `plots/active-hooks/hook-{名称}.md` 文件名（若已建立追踪文件）。

| 动作 | 钩子名     | 追踪文件           | 进展描述         |
| ---- | ---------- | ------------------ | ---------------- |
| 推进 | 母亲遗物   | hook-母亲遗物.md   | 主角首次展示玉佩 |
| 回收 | 小师妹下落 | hook-小师妹下落.md | 本章救出         |
| 引入 | 新敌人身份 | hook-新敌人身份.md | 本章埋下         |

**校验要点**：

- 若本章所属章数是 **5 的倍数**（5、10、15、20……），"引入"栏**至少有一条**（对应五章节点强制挂新钩子的红线）
- 每条"引入"或"推进"都要有对应的追踪文件路径（若暂未建立可标注"待建立"）
- 若钩子是**跨 5 章以上的新钩子**，必须同步新建追踪文件

### 模块 6 · 章节结构

**必填**：三段结构的字数占比与关键动作。

```markdown
### 凤头（0-10%，约 XX 字）

- 开篇方式与氛围：<是冲突动作切入，还是环境/回忆渲染开场？写出开篇规划>

### 猪肚（10-85%，约 XX 字）

- 关键情节与互动：<场景 1 → 场景 2 → 场景 3，叙述重点如何在情节中呈现>

### 豹尾（85-100%，约 XX 字）

- 收尾方式：<是制造悬念、情感留白还是自然停顿？写明具体规划，禁止空洞套话>
```

### 模块 7 · 触发标记与写作约束

**7a · 触发标记**（若本章包含以下场景之一，**必须**打上对应触发标记）：

- [ ] **战斗**（有真正的出手对抗，不仅仅是口舌之争）
- [ ] **突破**（境界 / 职业等级的跃升）
- [ ] **情感重头戏**（诀别 / 表白 / 撕破脸 / 深度共情）
- [ ] **大悲 / 大怒 / 大惧**（强烈情绪爆发）

触发标记会让质量质检分支 (quality-editor) 叠加对应的专项红线（见 [quality-checklist.md](quality-checklist.md) §二）。

**7b · 写作约束**（本章需要规避的红线）：

从项目红线文件中筛选出与本章相关的条目。示例：

```markdown
- 本章涉及 {某物品}，**严禁**写成"记录信息 / 秘密"的载体（项目红线：{某物品}性质）
- 本章涉及主角境界，**不得**把职业等级与境界硬性绑定（项目红线：职业-境界解耦）
- 本章是第 10 章，**必须**同步生成里程碑（5 章节点强制）
```

**校验要点**：

- 触发标记：蓝图作者必须主动勾选（若本章实际有战斗但未标记 → 质检时 FAIL）
- 写作约束：若项目红线文件存在，本节至少应列出 1-2 条与本章相关的红线
- 若项目红线文件不存在，写作约束可写"无特殊红线，仅遵循通用规则"

### 模块 8 · 上下文准备

**8a · 人物状态**：本章开始前与结束后的关键人物状态差异（只写有变化的角色）。

```markdown
### 主角

- 章前：修为 XX，带伤 X 处，与 X 的关系为 XX
- 章后：<关键变化>
- 认知变化：本章主角意识到 XX

### 核心配角 X（仅有变化时写）

- 章前：...
- 章后：...
```

**8b · 体系文档引用清单**：本章涉及哪些体系文档，列出**具体文件名**与**引用点**。

| 文件名      | 引用点          | 用途         |
| ----------- | --------------- | ------------ |
| 修炼体系.md | #筑基期         | 主角境界参考 |
| 人物体系.md | #主角 / #小师妹 | 人物背景     |

**校验要点**：

- 主角状态必有
- 体系文档引用不得为空（本章一定会涉及至少一份体系文档）
- 人物状态必须能与最新里程碑的"人物状态"衔接，不得断档
- 日常章若角色无变化，可写"同前章，无变化"（简化处理）

## 三、蓝图完备性校验规则

质量质检分支 (quality-editor) 或蓝图校验器在以下场景必须跑完备性校验：

### 3.1 触发时机

| 时机                     | 行为                               |
| ------------------------ | ---------------------------------- |
| 卷规划工作流生成新蓝图后 | 立即校验，不合格阻塞该批次蓝图交付 |
| 章节创作工作流前置检查时 | 再次校验，不合格阻塞正文创作       |

### 3.2 校验项（逐项 PASS / FAIL）

| #   | 检查项                                                        | 判定                                          |
| --- | ------------------------------------------------------------- | --------------------------------------------- |
| B1  | 8 个模块是否齐全？（按标题识别）                              | 缺任一 → FAIL                                 |
| B2  | M1 · 叙事目标：是否为一句话、只一件事？                       | 塞多件 → FAIL                                 |
| B3  | M3 · 情绪节奏：位置点是否 ≥ 3？                               | < 3 → FAIL                                    |
| B4  | M2 · 场景设计：场景名/地点/出场角色/核心事件是否齐全？        | 必填字段有空 → FAIL                           |
| B5  | M4 · 叙事重点：是否非空且包含描写？                           | 为空或无内容 → FAIL                           |
| B6  | M5 · 钩子进展：若是 5 章节点，"引入"栏是否 ≥ 1 条？           | 5 章节点未引入 → FAIL                         |
| B7  | M6 · 章节结构：凤头与豹尾是否写了具体的展现方式（拒绝套话）？ | 为空或出现“一切才刚开始”等空洞总结套话 → FAIL |
| B8  | M8 · 上下文准备：体系文档引用是否非空？                       | 空清单 → FAIL                                 |

### 3.3 校验失败的处理

- 第一次不合格：返回明确错误清单（"模块 X 缺失"、"记忆点为空"等），让用户 / 下一轮模型修订
- 连续 2 次不合格：停止自动校验，要求用户人工介入
- 通过后标记为"蓝图已审核 · PASS"，进入下一阶段

## 四、完整示例蓝图（通用化，不绑定具体项目）

> 以下示例使用通用化的角色与场景，适用于任何修真 / 玄幻项目。

```markdown
---
name: 第13章蓝图·血夜·石刻残影
type: chapter_blueprint
volume: 1
chapter: 13
target_words: 3800
status: draft
tags: [blueprint]
---

## 1. 基础信息与目标

- **所属 Arc**：Arc 3 · 危机
- **情绪基调**：紧张、压抑、虐心
- **叙事目标**：主角独自潜回旧居，在血夜中找到母亲留下的石刻残片，并与前来灭口的敌人擦肩而过。

## 2. 场景设计

### 场景 1：血夜旧宅

- **地点**：永宁城北 · 旧宅遗址
- **出场角色**：主角（独自）
- **核心事件**：夜潜旧居外围，发现灵脉已被人布设血阵
- **冲突类型**：人物 vs 环境（血阵压制）+ 人物 vs 命运（母亲留下的谜）
- **推进钩子**：母亲遗物（hook-母亲遗物.md · 推进）

### 场景 2：主屋暗格

- **地点**：永宁城北 · 旧宅主屋 · 地下暗格
- **出场角色**：主角
- **核心事件**：破解三层禁制，取得母亲留下的石刻残片
- **冲突类型**：人物 vs 自我（恐惧与决心的拉扯）
- **推进钩子**：母亲遗物（推进）+ 永宁灭门真相（hook-永宁灭门真相.md · 首次推进）

### 场景 3：生死一线

- **地点**：旧宅主屋 · 暗格封闭瞬间
- **出场角色**：主角 + 未露面的敌人（仅脚步声与一句呼唤）
- **核心事件**：敌人抵达主屋，主角躲入暗格，封闭前一刻听见自己的本名

## 3. 情绪节奏

| 位置    | 情绪 | 强度 | 内容概述                           |
| ------- | ---- | ---- | ---------------------------------- |
| 0-10%   | 紧张 | 8    | 血夜笼罩旧宅，异变突起             |
| 10-40%  | 压抑 | 6    | 摸进残破主屋，层层禁制破解         |
| 40-70%  | 爆发 | 9    | 找到石刻，认出母亲笔迹             |
| 70-90%  | 凝滞 | 7    | 敌人脚步逼近，躲进暗格             |
| 90-100% | 悬念 | 8    | 暗格封闭一瞬，外面有人叫出他的本名 |

## 4. 记忆点

1. **50% · 情感爆发**：主角触摸石刻认出母亲笔迹，情感决堤；以身体细节与意识浮现代替直白哭诉
2. **70% · 智谋博弈**：主角提前取了符纸压制呼吸与灵力波动，借此骗过敌人神识扫描
3. **95% · 认知反转**：本以为敌人只是追石刻；豹尾揭示对方知道主角真实身份

## 5. 钩子进展

| 动作 | 钩子名       | 追踪文件                                | 进展描述         |
| ---- | ------------ | --------------------------------------- | ---------------- |
| 推进 | 母亲遗物     | plots/active-hooks/hook-母亲遗物.md     | 取得石刻残片     |
| 推进 | 永宁灭门真相 | plots/active-hooks/hook-永宁灭门真相.md | 首次出现真相碎片 |
| 引入 | 神秘敌人身份 | plots/active-hooks/hook-神秘敌人身份.md | 敌人知道主角本名 |

## 6. 章节结构

### 凤头（0-10%，约 380 字）

- 开场：血色月光下的废宅，异象先行——有人提前布好了等待的局
- 主角入场即遭遇异变（青砖异常），第一个动作就是危险信号
- 禁止环境描写开局 ✅（异象与主角动作同步出现）

### 猪肚（10-85%，约 3000 字）

- 情绪曲线：压抑（破解禁制） → 爆发（认出笔迹） → 凝滞（躲入暗格）
- 关键场景：血阵压制 → 破解三层禁制 → 石刻现身 → 敌人脚步逼近
- 记忆点爆发：50% 认字、70% 屏息骗神识

### 豹尾（85-100%，约 420 字）

- **新变量**：暗格封闭的最后一瞬，主角清楚听见外面的敌人叫出他母亲曾给他起的小名
- **悬念类型**：身份反转（敌人是旧识）+ 认知翻转（母亲的圈子比主角想象更大）

## 7. 触发标记与写作约束

### 触发标记

- [x] 情感重头戏（场景 2 石刻认字）
- [x] 大悲 / 大怒 / 大惧（悬念段：听到自己本名的惊惧）
- [ ] 战斗
- [ ] 突破

### 写作约束

- 本章涉及母亲遗物，**严禁**把石刻写成"记录所有隐秘真相的账本"；石刻只是母亲留下的**家族心法片段 + 一个地名**
- 本章使用符箓，不得让主角以筑基初期修为炼制**高阶符箓**；只能使用现成中阶符纸
- 豹尾情绪用"听见本名"的**一句话**收锤，禁止长篇解释

## 8. 上下文准备

### 人物状态

**主角**

- 章前：筑基初期，轻伤未愈，决定独自潜回旧宅
- 章后：筑基初期（无变化），新添灵力透支导致轻微内伤，获得**母亲石刻残片**
- 认知变化：首次意识到"敌人不只想杀他，还早就认识他"

### 体系文档引用

| 文件名      | 引用点             | 用途              |
| ----------- | ------------------ | ----------------- |
| 修炼体系.md | #筑基期 · 真元运用 | 主角灵力压制      |
| 功法体系.md | #禁制破解 · 中阶   | 场景 2 禁制破解   |
| 符箓体系.md | #屏息符            | 记忆点 2 用的符纸 |
| 地理体系.md | #永宁城            | 场景地点设定      |
| 人物体系.md | #主角 / #主角母亲  | 人物背景          |
```

## 五、小结

- 8 个模块缺一不可，顺序与命名固定
- 蓝图完备性校验是**双阶段闸门**（批次蓝图生成后、章节创作前）
- 豹尾"新变量"必须具体，不能只写"悬念"

- 记忆点 ≥ 1（允许伏笔、情感锚点等非"爽"类记忆点）
