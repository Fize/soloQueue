# 质量检查清单（红线 + 黄线）

> 这是内建的**质量质检分支 (quality-editor)** 的**执行清单**。**章节创作分支 (story-engine)** 产出章节正文后，必须**逐项**跑完本清单。
> 红线项任一 FAIL → 拒绝交付，回到章节创作分支 `story-engine` 修订，最多 2 轮。
> 黄线项不阻断交付，但必须写入质检报告。
>
> 项目红线外挂文件可以**追加**红线 / 黄线项（见 [project-override-spec.md](project-override-spec.md)），但不得**删除**本清单的任何通用项。

---

## 一、红线项（通用 · 10 项 · 任一 FAIL 即拒收）

### R0 · 蓝图无正文句（蓝图阶段专用）

> 此项仅在**蓝图交付审核**阶段执行（卷规划后 / 章节创作前置检查）。

- 蓝图中**不得**出现具体对话内容（引号包裹的台词）、正文级别的描写句
- 蓝图只描述目的、节奏、场景结构和情绪走向，不预写正文
- **违规示例**：爽点描述里出现 `"他低声道：……"` 或凤头里出现具体开场句子
- **合规示例**：`情感爆发，以身体细节代替言语；主角用符纸压制灵力波动骗过神识`
- 出现一次即 FAIL，返回蓝图修订

### R1 · 零禁用词

- 检查正文中是否出现 [anti-ai-patterns.md](anti-ai-patterns.md) §一 的 12 条禁用词
- **出现一次即 FAIL**
- 禁用词清单：需要强调的是 / 总的来说 / 值得注意的是 / 从某种意义上说 / 不可否认 / 可以这样理解 / 某种程度上 / 不难看出 / 综上所述 / 一言以蔽之 / 换言之 / 毫无疑问

### R2 · 段落不超过 8 行

- 扫描每段，任一段落超过 8 行 → FAIL
- 6-8 行：黄警（不阻断）
- 对话段可以单行 / 双行

### R3 · 开篇（凤头）合规

- 开篇设计契合叙事氛围与人物当前状态
- **禁止**干瘪枯燥的说明性背景介绍开篇（如“话说在XX世界，有一座XX山……”），描写和回忆需自然融入场景与动作
- 违反即 FAIL

### R4 · 收尾（豹尾）合规

- 收尾顺应故事逻辑自然落锤，不强求每章刻意制造悬念断章
- **禁止**空洞、公式化的总结性套话（如“这一夜，注定不平静”、“一切才刚刚开始”）
- 违反即 FAIL

### R5 · 关键场景五感 ≥ 3

- 关键场景（战斗 / 突破 / 危机 / 诀别 / 重大发现）的五感描写必须 ≥ 3 种
- 详见 [anti-ai-patterns.md](anti-ai-patterns.md) §三
- 不足即 FAIL

### R6 · 设定解说 ≤ 10%

- 机械式大段设定解说占比不得超过章节字数的 10%
- 检查方法：挑出"段落整段都在解释规则 / 体系 / 背景"的块，累加字数 / 总字数
- 超过 10% 即 FAIL

### R7 · 叙事重点聚焦

- 每章需具有符合故事逻辑的重点描写（如人物关系推进、情感进展、冲突呈现、动作细节或核心线索），禁止全篇毫无重点的流水账
- 不足即 FAIL

### R8 · 五章节点里程碑含"阶段成果 + 下一阶段钩子"

- 若本章是第 5、10、15、20…… 章：
  - 必须生成对应 `plots/milestones/vol-{N}-ch-{Y}-summary.md`
  - 里程碑必须**同时包含**"阶段成果（A 节）"和"下一阶段入口（B 节）"（见 [milestone-rules.md](milestone-rules.md)）
  - 缺任一节 → FAIL
  - 本章豹尾的新变量必须与里程碑 B 节呼应

### R9 · 无现代词汇污染（修玄题材）

- 扫描正文，禁止出现以下现代词：
  - 时间：秒、分钟、小时
  - 度量：毫米、厘米、米、千克
  - 科技：计算机、数据库、网络、信号、系统
  - 商业：订单、客户、业绩、KPI、营业执照、税务、流程、效率、团队、数据
- 出现一次即 FAIL
- 其他题材可通过项目红线文件覆盖此项

### R10 · 融合笔法合规（沉默见证者）

> 依据 [fusion-brushwork.md](fusion-brushwork.md)，逐项检查：

**R10a · 内心独白禁令**（§3.1 · 零容忍）：

- 扫描正文，禁止出现以下心理活动标记词：
  - 他想 / 他感到 / 他意识到 / 他回忆起 / 他觉得 / 他认为 / 他明白
- 出现一次即 FAIL
- 心理活动一律转化为外部细节（身体反应、动作、环境变化）

**R10b · 叙述者不越界解读**（§一 + §三 · 零容忍）：

- 叙述者不得对可见动作赋予"含义"或"角色定义"
- **违规句式**：
  - "X 就是 Y"（如"树就是他的对手"）
  - "一个人把 XX 收在 XX 里，走出来的路就是这样的"（将动作翻译为本质）
  - "这是 XX 的默契/习惯/方式"（替读者总结模式）
  - **"像是/仿佛/好像 + 动作含义解读"**（如"像是这事已经完了""仿佛什么都没发生""像是连着别的什么东西一起吞进去的"——用比喻句型替读者解释动作的意思）
  - **"带着 XX 的 + 抽象态度"**（如"带着漫不经心的分量""带着不容置疑的威压"——给物理动作附加叙述者评判的情绪/态度标签）
  - **推断性措辞**：叙述者使用"大概""应该""或许""恐怕""想必"对可见事实做因果推理（如"大概是从老者手里夺的"——读者看见茶壶在爪牙手里就够了）
  - **"——"解释性收尾**：画面展示完毕，破折号后紧跟叙述者注解——价值判断（"不是什么值钱物件，被他护得比命还紧"）、动机猜测（"又觉得冒失"）、情绪命名、原因说明。破折号后只允许跟**新的具体信息**（新动作/新画面/转折），不允许跟**对前文的总结/评价/解释**
  - **"不是X，(是)Y"的变体**：除直接形式外，还包括"没看X，看的是Y""不在X，而在Y""并非X，而是Y"——结构等价，用否定+纠正替读者做阅读理解
  - **Callback提示**：叙述者主动提醒读者"跟昨天/上次/之前/在XX处……一样/一模一样"——读者自己会认出的 callback 才有力量，叙述者不应替读者发现
  - **对人物行为贴动机/情绪标签**：叙述者在动作描写后紧接动机判断（"觉得冒失""感到不妥""意识到不妙""明白过来"）——内心活动一律通过外部细节折射
- **判断标准**：这句话是在"记录可见的动作/细节"，还是在"告诉读者这个动作意味着什么"？后者即 FAIL。**"像是"不能作为免责前缀**——"他转身走了，像是不在乎"和"他转身走了，他不在乎"本质相同。
- **比喻 vs 含义解读的区分**：纯比喻描摹动作/状态的**可见质感**（"整条胳膊垂下去，像线被人剪断了"）→ 合规。含义解读用"像是"解释动作/声音/表情**背后的意思**（"像是连着别的什么东西一起吞进去的"）→ FAIL。判断方法：删掉"像是"部分后，剩余画面是否自足？
- **破折号自查**：搜索正文中所有"——"，逐一检查破折号后是**新信息**（新动作/新画面/转折）还是**对前文的解释**（评价/动机/原因/总结）。解释性的全砍。

**R10c · 叙述距离一致**（§三）：

- 不得出现"永远""总是""从 XX 起就"等跨时间总结——这些是叙述者跳出当前场景做的归纳
- 只写此刻可见的、具体的

## 二、触发标记专项红线（按蓝图标记叠加）

> 蓝图中的"触发标记"字段（M7）会激活对应专项红线，与本节通用红线**叠加**执行。

### T1 · 战斗触发 → 打斗场景专项

若蓝图标记了"战斗"，则额外检查 [combat-scene.md](combat-scene.md) §8.1 的红线项：

- 境界差异是 Show 而非 Tell
- 实力差距通过具体描写展示
- 五感 ≥ 3 种（与 R5 叠加）
- 越级挑战有合理支撑
- 打斗结果合乎境界逻辑

### T2 · 突破触发 → 突破合理性专项

若蓝图标记了"突破"，则额外检查：

- 有前兆（近几章有铺垫）
- 有积累（前期修炼 / 机缘 / 心境）
- 有契机（具体触发事件）
- 突破后不能立刻无敌（需要磨合期）

### T3 · 情感重头戏触发 → 情感专项

若蓝图标记了"情感重头戏"，则额外检查：

- 情感表达方式与角色性格、身份、背景完全一致（内敛者含蓄暗示，热烈奔放者应当直白坦荡，严禁因生套“含蓄”公式导致人物 OOC）
- 情感表达通过具体的语言、动作和场景细节来呈现（拒绝空洞直白的感情宣告，如“他们相爱了”）
- 有**代价**与张力（情感戏推动情节或人物成长，而非单纯的无摩擦发糖）

### T4 · 大悲 / 大怒 / 大惧触发 → 对话克制专项

若蓝图标记了"大悲 / 大怒 / 大惧"，则额外检查：

- 对话**克制字数**，宁短勿散
- 避免长篇解释
- 与身体反应联动（呼吸、指节、眼神、喉头、步伐、出手节奏）
- 详见 [writing-style.md](writing-style.md) §五.3

## 三、黄线项（通用 · 不阻断但必须写入报告）

### Y1 · 对话情绪纹理

- 对话有没有传递情绪？
- 不同情境下同句话有没有不同质地？
- 压着说 / 忍着说 / 硬撑着说 / 失控地说是否写出差别？

### Y2 · 比喻 / 意象的第一眼可读性

- 比喻读者第一眼能看懂本体吗？
- 是不是"作者自觉有味、读者第一眼看不懂"的那种？

### Y3 · 同角色说话方式一致性

- 本章中同一角色的说话方式是否与性格、关系、场景压力匹配？
- 是否与前几章该角色的语言风格一致？

### Y4 · 修玄身份言行一致性

- 修真者对话是否符合修为境界？
- 宗门弟子是否体现宗门文化？
- 散修是否体现江湖阅历？
- 凡人是否不越界干预修真问题？

### Y5 · 打斗中的智慧博弈层次

- 主角打斗是否体现冷静分析、临场应变？
- 是否用智谋弥补实力差距？
- 战术是否至少到第二层（功法相克 / 法宝制衡）？
- 详见 [combat-scene.md](combat-scene.md) §六

### Y6 · 段落 6-8 行警告

- 段落长度在 6-8 行的 → 黄警
- 不阻断交付，但报告中列出

### Y7 · 人物状态快照衔接

- 本章开头是否与上章末尾的人物状态衔接？
- 是否与最新里程碑的"人物状态"一致？

### Y8 · 钩子推进可视性

- 本章推进的活跃钩子，读者是否能感知到"有事在发生"？
- 避免钩子埋了却没体现

## 四、质检执行流程

```
1. 加载通用红线（R1-R10）+ 加载 fusion-brushwork.md（R10 依据）
   ↓
2. 加载项目红线追加项（来自项目红线外挂文件）
   ↓
3. 读取蓝图 M7 触发标记 → 加载对应专项红线（T1-T4）
   ↓
4. 逐项执行红线检查
   ↓
5. 任一 FAIL → 生成报告 + 触发章节创作分支 `story-engine` 回修（最多 2 轮）
   ↓
6. 红线全 PASS 后，逐项执行黄线检查
   ↓
7. 生成完整质检报告
```

## 五、质检报告模板

报告路径：`quality-control/correction-suggestions/vol-{N}-ch-{X}-quality-report.md`

```markdown
# 质量检查报告 · 卷 {N} · 第 {X} 章

## 总览
- **总分**：X/10
- **红线结果**：PASS / FAIL（回修第 N 轮）
- **黄线结果**：Y 项需注意

## 红线项清单

| # | 项目 | 状态 | 位置 | 问题 | 修正建议 |
|---|---|---|---|---|---|
| R1 | 零禁用词 | PASS / FAIL | 第 X 段 | 出现"值得注意的是" | 改为直接陈述 |
| R2 | 段落 ≤8 行 | PASS | - | - | - |
| ... | ... | ... | ... | ... | ... |

## 触发标记专项（如有）

| # | 项目 | 状态 | 问题 | 修正建议 |
|---|---|---|---|---|
| T1 | 战斗场景专项 | PASS / FAIL | ... | ... |

## 黄线项清单

| # | 项目 | 状态 | 位置 | 问题 | 建议 |
|---|---|---|---|---|---|
| Y1 | 对话情绪纹理 | 需注意 | 第 X 段 | 只有"他很愤怒" | 改用动作展示 |

## 项目红线追加项（如有）

| 项目 | 状态 | 问题 | 修正建议 |
|---|---|---|---|
| （由项目红线文件注入） | PASS / FAIL | ... | ... |

## 结论

- [ ] 红线全 PASS → 章节可交付
- [ ] 红线有 FAIL → 触发回修（第 N 轮，最多 2 轮）
- [ ] 已 2 轮仍 FAIL → 提示用户人工介入
```

---

## 六、回修规则

- 红线 FAIL 第 1 轮：自动回到章节创作分支 `story-engine` 修订
- 红线 FAIL 第 2 轮：再次自动回修
- 红线 FAIL 第 3 轮（即回修 2 轮后仍失败）：**停止自动回修**，生成报告并提示用户人工介入
- 每轮回修后必须重新跑一次完整红线清单（含触发标记专项）
