# 去 AI 痕迹规范（通用）

> 本文件定义"写出来不像 AI"的硬性约束。所有条目默认为**红线级**（零容忍），违反即必须修正。
> 红黄线清单的汇总见 [quality-checklist.md](quality-checklist.md)。

---

## 一、禁用词（红线 · 零容忍）

以下 AI 惯用的总结性/说教性句式、时间过渡模板以及情绪动作套路，**严禁出现在正文**（蓝图/里程碑等策划文档可以酌情使用，但正文出现一次即为红线 FAIL）。

这些硬性规则已接入静态分析工具 `scripts/novel_lint.py`。在提交正文前，必须先运行该脚本进行程序化静态扫描。

| 类别 | 禁用词/变体句式 | 替代思路 / 违规说明 |
| --- | --- | --- |
| **转折说教类** | `需要强调的是`、`总的来说`、`值得注意的是`、`值得一提的是`、`从某种意义上说`、`从某种意义上来说`、`不可否认`、`不可否认的是`、`可以这样理解`、`某种程度上`、`不难看出`、`综上所述`、`一言以蔽之`、`换言之`、`换句话说`、`毫无疑问`、`毫无疑问的是`、`不得不说`、`不得不承认`、`不得不置信`、`说实话`、`简而言之`、`简单来说`、`总而言之`、`由此可见`、`显而易见`、`与此同时的是`、`也就是说` | 直接把事实写出来，或让人物动作/对白展示。严禁叙述者跳出来向读者进行总结和说明。 |
| **时间流逝套路** | `随着时间的推移`、`时光荏苒`、`光阴似箭`、`在接下来的时间里`、`在接下来的日子里`、`接下来的剧情中`、`随之而来的是`、`随之而来的` | AI 在切换场景或转场时极度依赖此类模板化词汇。应直接通过具体事物的变化或起承转合直接描写时间流逝。 |
| **动作神态套路** | `微微一笑`、`嘴角微微上扬`、`嘴角微扬`、`露出一抹笑意`、`淡然一笑`、`会心一笑`、`深吸一口气`、`深吸气`、`吸了一口气`、`闪过一丝`、`掠过一丝`、`浮现出一丝`、`不仅如此`、`不仅仅是`、`不禁`、`忍不住` | AI 表达人物神态、动作以及内心波动的固定套路，出现频率极高。应以具体、独特的肢体语言、微表情或心理细节替换。 |
| **解释/纠正句式** | `不是...而是...`、`并非...而是...`、`并不是...而是...`<br>`与其说是...不如说是...`<br>`不在于...而在于`、`不在...而在`<br>`没看...看的是`、`没有看...看的是`<br>`不是为了...而是为了`<br>`非但没有...反倒`、`非但不是...反而是`<br>`不单单是...更是`、`不单是...更是` | **R10b 叙述者解释性句式。** 画面已经 Show 了，严禁叙述者再用否定+纠正的句式替读者做阅读理解。只留画面本身。 |

> 项目可通过"项目红线外挂文件"**追加**禁用词，但不得**删除**上述通用禁用词。加载协议见 [project-override-spec.md](project-override-spec.md)。

## 二、Show-not-Tell 原则（红线）

> 最常见的 AI 痕迹：**直接告诉读者情绪/实力/强度**，而不展示出来。

### 2.1 错误 vs 正确对照

| ❌ Tell（错误） | ✅ Show（正确）                                |
| --------------- | ---------------------------------------------- |
| 他非常愤怒      | 他的拳头攥得咯咯作响，青筋爬满太阳穴           |
| 这个敌人很强    | 脚下的青石寸寸碎裂，草木枯萎倒伏               |
| 她很伤心        | 她低头去拎茶壶，手腕僵了一下，壶盖碰出一声轻响 |
| 场面很压抑      | 连风都停了，侍女们连呼吸都屏住                 |
| 他下定决心      | 他把那枚玉佩慢慢收进袖中，扣上暗扣             |

### 2.2 Show 的三个抓手

1. **环境反应**：用环境的变化展示力量/情绪
2. **身体小动作**：指节、呼吸、眼神、喉头、步伐
3. **他人反应**：旁观者的反应倒映主角状态

### 2.3 禁止解释画面（红线）

> 这是 "Show-not-Tell" 最常见也最隐蔽的变体：**画面已经 Show 出来了，作者立刻跳出来解释它的含意/原因/时长。**

- Show 完立刻 Tell，等于作者替读者做阅读理解。读者自己填出来的，比作者告诉他的更有力
- **严禁**"不是X，是Y"句式——先否定一个错误解读，再给出正确解读。这是作者在替读者思考
- **严禁**画面后紧跟解释性破折号（"——不是今天才有的，是日复一日……"）
- **严禁**人物动作后紧跟动机解说（"她不让丫鬟替他摆碗。不是丫鬟做不好，是她想做。"）
- **严禁**情绪画面后紧跟情绪命名（"她眼睛里带着笑。不是开心，是满足。"）
- **严禁**叙述者做推断性措辞（"大概是从老者手里夺的""应该是不愿意说"——叙述者不是侦探，只记录可见事实）
- **严禁**叙述者主动提醒读者发现 callback（"跟昨天在茶摊说的一模一样""跟上次一样"——读者自己会记住，不需要提示）
- **严禁**叙述者对人物行为贴动机/情绪标签（"又觉得冒失，缩回去了"——"缩回去了"已经 Show 了，"觉得冒失"是替人物补内心）

**违规示例**：

| ❌ Show 完立刻 Tell                                                | ✅ 只留画面                              |
| ------------------------------------------------------------------ | ---------------------------------------- |
| 砖面上有一片深色湿痕——不是今天早上才有的，是日复一日汗渗进砖缝里。 | 砖面上有一片深色湿痕。                   |
| 脚步声很稳，不是迟疑，是从容。                                     | 脚步声很稳。                             |
| 背永远挺得笔直——不是刻意端着，是多年练武骨头习惯了。               | 背永远挺得笔直。                         |
| 她不让丫鬟替他摆碗。不是丫鬟做不好，是她想做。                     | （丫鬟在廊下候着，没有上前。）           |
| 她眼睛里带着一种很淡的笑——不是开心，是满足。                       | 她眼睛里带着一种很淡的笑。               |
| 这是父子之间的默契。                                               | （删掉。前文的动作已经传达了。）         |
| 一个手里捏着只紫砂茶壶，壶把磨出了包浆。大概是从老者手里夺的。     | 一个手里捏着只紫砂茶壶，壶把磨出了包浆。 |
| 四个字——跟昨天在茶摊说的一模一样。                                 | （删掉。读者自己会认出这四个字。）       |
| 拍了两下又觉得冒失，缩回去了。                                     | 拍了两下，缩回去了。                     |
| 不是什么值钱物件，被他护得比命还紧。                               | （删掉。布包打了三层结已经传达了。）     |
| 他拇指摩挲着碗沿的缺口，跟昨天在小酒摊上一样。                     | 他拇指摩挲着碗沿的缺口。                 |
| 像是连着别的什么东西一起吞进去的。                                 | （删掉。吞咽声音变重已经传达了。）       |

**自查**：搜索正文中的破折号"——"和"不是"——如果"——"后面跟的是解释而非新信息，砍。如果"不是"后面跟的是纠正读者理解的话，砍。搜索"大概""应该""或许""恐怕""想必"——逐一确认是否叙述者在替读者补推理。搜索"跟XX一样""和上次一样"——逐一确认是否叙述者在替读者发现 callback。搜索人物动作后紧跟的"觉得""感到""意识到""明白"——叙述者是否在替人物补内心活动。

## 三、五感描写（红线）

| 场景类型                                         | 五感最低要求           |
| ------------------------------------------------ | ---------------------- |
| **关键场景**（战斗、突破、危机、诀别、重大发现） | **≥ 3 种感官**（红线） |
| 普通场景                                         | ≥ 2 种感官             |
| 日常过渡                                         | 不强制                 |

### 3.1 五感清单

- **视觉**：光、影、色彩、动作轨迹、残影
- **听觉**：爆鸣、金铁交击、破空声、喘息、沉默
- **触觉**：气浪、震动、冷热、衣料摩擦、肌肤触感
- **嗅觉**：血腥气、烧焦味、草木香、熏香、汗味
- **味觉**：铁锈味、苦涩、甘冽、药香
- **额外可用**（玄幻/修真题材）：灵压、神识冲击、道韵、因果感知

### 3.2 常见错误

- 整段只有视觉：加入听觉与触觉
- 感官描写堆砌在一起：打散分布到段落不同位置
- 感官与情绪脱节：五感应服务于情绪，不是罗列

## 四、设定解说限制（红线 · 占比）

- **机械式大段设定解说占比 ≤ 10%**（以章节字数为分母）
- 禁止"开局千字设定背景板"
- 把设定**拆碎融入**对话、动作、环境、人物反应中

### 4.1 典型违规

- 章节开头连续三段解释世界观规则
- 战斗中突然插入两段解释法术原理
- 对话里一个配角给主角（和读者）讲了一整套体系

### 4.2 合规做法

- 设定碎片随情节自然引出，每次只露一角
- 让设定通过**代价**体现（主角用了某功法，付出了什么）
- 关键设定延后揭示，制造"原来如此"的爽感

## 五、用词设计（红线 · 题材相关）

> 本节约束默认对**修真/玄幻/古代仙侠**题材生效。现代/都市/科幻题材可通过项目红线外挂文件覆盖。

### 5.1 避免现代词汇污染（修玄题材红线）

| ❌ 现代词            | ✅ 修玄替代                            |
| -------------------- | -------------------------------------- |
| 秒、分钟、小时       | 瞬息、弹指、一炷香、半个时辰、一个时辰 |
| 毫米、厘米、米       | 几寸、数尺、丈许                       |
| 克、千克、吨         | 几斤、数两、若干石                     |
| 计算机、数据库、网络 | 不应出现，换成传音符、观星台、天机盘等 |
| 订单、客户、业绩     | 不应出现，换成买主、香客、门楣         |
| 营业执照、税务       | 不应出现，换成令牌、贡金               |
| 信号、系统           | 不应出现，换成感应、道则               |
| 效率、流程、团队     | 改成火候、章程、同门                   |
| 数据、概率           | 改成天数、几率（慎用）、造化           |

### 5.2 保持语言风格一致

- 所有文档（正文、蓝图、里程碑、钩子）统一使用题材语言
- 禁止出现武侠小说中常见的过时身份标签（如"江湖侠客"混入修真世界）

## 六、身份言行规范（题材红线 · 默认修玄）

> 本节定义**修真/玄幻**题材下"角色身份与言行必须一致"的硬性要求。其他题材由项目红线外挂文件替换。

### 6.1 角色身份与言行一致性

- 修真者的言行必须符合其**修为境界和身份地位**
- 低阶修士对高阶修士必须体现**敬畏**（如称"前辈"、"真人"、"道友"视修为差距而定）
- 同阶修士相称须符合惯例（道友、师兄、师弟等）
- **严禁**修真者使用凡人市井语言或现代口语

### 6.2 场景设定规范

- 修真场景必须符合修真世界观（洞府、灵脉、宗门、秘境）
- 禁止出现凡人世界场景元素（现代建筑、科技产物、凡俗商业术语）
- 日常生活场景须**修真化**（以灵石交易、以传音符通讯、以飞剑代步）
- 修行场所须体现修真特征（聚灵阵、禁制、法宝陈列）

### 6.3 对话语言规范

- 修真者对话须体现修为境界（高阶言简意深、低阶恭敬谨慎）
- 宗门弟子对话须体现宗门文化和等级制度
- 散修对话须体现江湖阅历和生存智慧
- **严禁**以下类型内容：
  - 凡人市井俚语
  - 现代商业术语
  - 凡俗官府用语
  - 现代科技概念

### 6.4 身份转换注意事项

- 角色在凡人界行走时，可适度收敛修为，但**本质仍是修真者思维**
- 不得以凡人逻辑解决修真问题（用"讲道理"代替实力说话）
- 涉及凡人时，须体现修真者与凡人的**本质区别**（寿命、力量、眼界）

## 七、AI 自查清单（创作后必过）

写完一章或一段后，逐项自查：

**基础红线**：

- [ ] 有没有出现第一节 12 条禁用词？（任一出现即改）
- [ ] 有没有 Tell 式句子？（"他很 X""她非常 Y""场面很 Z"）（有即改）
- [ ] Show 完有没有紧跟解释？（搜索"——不是""不是X，是Y""这是XX的默契"等解释句式，有即砍）
- [ ] 关键场景五感够 3 种了吗？（不够即补）
- [ ] 设定解说占比超过 10% 了吗？（超即拆碎或删减）
- [ ] 题材用词有污染吗？（修玄题材检查 5.1 清单）
- [ ] 角色身份言行一致吗？（修真者有没有说现代口语）

**融合笔法专项**（依据 fusion-brushwork.md）：

- [ ] 有没有"他想 / 他感到 / 他意识到 / 他回忆起"？（内心独白禁令，有即转为外部细节）
- [ ] 叙述者有没有替画面下定义？（"X 就是 Y""一个人把 XX 收在 XX 里"等——有即砍）
- [ ] 有没有"永远""总是""从 XX 起就"等跨时间总结？（有即改为此刻所见）
- [ ] 情感有没有通过具体物件和动作折射？（而非直接描述情感状态）

任意一项失败，**禁止提交章节**，回到章节创作分支 `story-engine` 修订。
